chrome-use

chrome-use 让任意 AI agent 直接驱动你已登录的真实 Chrome—— Claude Code、Cursor、Codex、你自己的脚本。它复用你现成的登录会话， 对反爬完全不可检测，因为它就是你的真实浏览器： CreepJS 实测 0% 机器人。

bash 
# 一行装好 CLI —— 无 npm、无 token
$ curl -fsSL https://raw.githubusercontent.com/leeguooooo/chrome-use/main/install.sh | sh
✅ 第一次来？
先读 安装 装上浏览器扩展并连你的真实 Chrome，再走一遍 核心循环（snapshot → find → act）——这就是你用 chrome-use 干活的 90%。

为什么用 chrome-use

不用开新 Chrome，不用重新登录，不用跟「你是不是机器人」较劲。常规浏览器自动化 开的是一个空 profile 的全新浏览器——每个登录你重做，每个验证码你硬扛，最后还被网站标成 自动化。chrome-use 直接把 agent 指向你现在正用着、已经登录了所有网站的那个 Chrome。它在 你的窗口里点击，你看着它干活，撞到 2FA 或验证码的瞬间你接管一下，它接着跑。

🔒 你真实、已登录的 Chrome

cookies、登录态、指纹全是真的——因为它就是你的真实浏览器。对反爬系统完全不可检测。
🧩 走扩展，不走调试端口

一键商店扩展 + 原生消息，不走 --remote-debugging-port，所以永远不弹「Allow remote debugging?」。
🎨 多 agent 隔离

每个 --session 拿到自己彩色的标签组，多个 agent 共用一个真实 Chrome 互不打架。
🕵️ 后台静默运行

全程在后台跑——新标签页不抢焦点，agent 永远不强制前置某个标签、不偷你正在看的前台窗口。

为什么不直接用……

  • Playwright / Puppeteer / browser-use？它们启动的是空浏览器——每个登录你重做、每个验证码你硬扛，最后还被识别为自动化。chrome-use 复用你已有的会话。
  • Claude 的 Chrome 插件？很好用，但只能给 Claude 用。chrome-use 给任意 agent / CLI 用，同一套真实浏览器。
  • 裸 --remote-debugging-port（CDP）？Chrome 136+ 每次连接都弹 「Allow remote debugging?」。chrome-use 永远不弹——一键商店扩展，走原生消息。
ℹ️ 拿数据说话
真实浏览器指纹拿去实测过最硬的公开检测器：CreepJS 0% 机器人 · 0% headless， 无 Runtime.enable CDP 泄漏（默认关），权限面极小—— 7 个权限，无 <all_urls>。详见 连接真实 Chrome 与反检测。

为什么它不会被封

很多网站主动封禁浏览器自动化，踩中反爬系统连账号一起封。chrome-use 的答案是把 反检测当成一等能力：它不去模拟一个真人——它就是你已登录的真实浏览器， 所以网站读它为 100% 真人，也就无从封起。

🛡️ CreepJS 0% 机器人 · 0% headless

走扩展连你的真实 Chrome 时不注入任何 JS 补丁——没有伪装，也就没有破绽给测谎器抓。bot.incolumitas / sannysoft / BrowserScan 全绿。
⚙️ 原生覆盖 > JS 谎言

--launch 独立浏览器用原生覆盖：navigator.webdriver=false（Emulation.setAutomationOverride）、timezone、WebRTC 本地 IP、canvas/audio 噪声——都不是能被反查的重定义 getter。headless 默认禁用（本身就是机器人特征）。
🖐️ --humanize 过行为检测

指纹之外，Akamai / PerimeterX / DataDome 还打行为分。开启后光标走曲线贝塞尔轨迹、落在元素内的抖动点、打字节奏可变——且自适应：命中这些厂商的页面自动升级，普通站保持瞬时零开销。
☁️ Cloudflare 一次通过复用

过一次挑战拿到的 cf_clearance（绑 IP+UA）被保存复用，不用每次重刷。随时 chrome-use stealth status 自查 webdriver / chrome / plugins / UA 探针。
