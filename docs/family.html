*-use 家族

chrome-use 只是一个更大想法里的一件。整个 *-use 家族 共用同一条哲学：不造模拟环境、不开虚拟机、不 spoof 指纹—— 直接接管你手里真实、已登录的那一个设备。浏览器、手机、密码库、账号会话， 每一件都是 agent 能安全调用的开源工具。

同一个思路：驱动真实设备

模拟环境总有裂缝：全新的浏览器指纹、没有登录态的会话、被反爬系统识别的自动化痕迹。 家族里的每件工具都换了个前提：它 就是 你本来在用的那台设备，所以 cookie、session、 登录态、指纹全都是天然真实的，agent 只是替你按下按钮而已。

✅ 共同点
每件工具都是本地运行、开源、面向 AI agent 设计的 CLI。它们可以单独用，也可以组合—— 比如 bitwarden-use 取出密码交给 chrome-use 登录， cookie-use 再把这份登录态存起来跨 profile 复用。

成员一览

每件工具一句话概括与真实仓库地址。想深入某一件，点开对应的 GitHub 仓库。

🌐 chrome-use

驱动你真实、已登录的 Chrome。共享现成会话，反爬系统检测不到——因为它就是你的真实浏览器。
📱 iphone-use

驱动你真实的 iPhone。操作没有 API 的 iOS App、导出手机上的数据、点按/输入/滚动/截图。
🔐 bitwarden-use

从你的 Bitwarden 密码库取出密码、2FA 和 passkey，让 agent 用账号密码登录。
🍪 cookie-use

加密的、站点无关的多账号会话保险库：保存、切换、注入某个网站的多个登录态。

chrome-use — 驱动真实 Chrome

家族里的主力。它把任意 agent（Claude Code、Cursor、Codex 或你自己的脚本） 接到你已经在用的那个 Chrome 上——一个浏览器扩展加原生消息通道，不开调试端口、复用现成 cookie。 一行命令安装：

install 
# 从最新 GitHub Release 下载对应平台的预编译二进制
$ curl -fsSL https://raw.githubusercontent.com/leeguooooo/chrome-use/main/install.sh | sh

把技能包装进任意 agent 运行器（Claude Code 推荐 plugin marketplace——整个 *-use 家族都在 leeguooooo/plugins 一个市场里）：

skill 
# Claude Code
> /plugin marketplace add leeguooooo/plugins
> /plugin install chrome-use@leeguooooo-plugins
# 其他 runner
$ npx skills add leeguooooo/chrome-use

装上 Chrome Web Store 扩展后，一次性注册原生消息宿主，然后就能接管你真实的浏览器：

bash 
# 注册原生消息宿主（只需一次）
$ chrome-use extension install
$ chrome-use open https://x.com/home
✅ 想深入 chrome-use
从 概览 开始，或直接看 驱动真实 Chrome（扩展） 了解扩展 + 原生消息的接管原理。 想知道为什么要这么做，看 和无头方案的对比 与 反检测原理。

bitwarden-use + cookie-use — 让 agent 带着身份登录

两件与登录相关的工具，正好补上 chrome-use 的上游与下游。 bitwarden-use 从你的 Bitwarden 密码库取出密码、2FA 和 passkey， 交给 agent 完成真正的账号密码登录；cookie-use 则把登录态本身当成资产： 加密存进本地保险库，随时切换、跨 Chrome profile 注入。

在 chrome-use test 的测试套件里，你可以直接用 account: 注入一个 cookie-use 保存的登录态，让每条用例都从「已登录」状态开始：

smoke.yaml 
# smoke.yaml
suite: chatgpt smoke
setup:
  - account: chatgpt/huayue          # 注入一个 cookie-use 登录态（可选）
cases:
  - name: home loads logged in
    steps:
      - open: https://chatgpt.com/
      - wait: { load: networkidle }
    assert:
      - url: { contains: chatgpt.com }
      - visible: "#prompt-textarea"

iphone-use — 驱动真实 iPhone

有些数据只活在手机上：Apple Health、银行 App、没有开放 API 的 IM。 iphone-use 把同样的「接管真实设备」思路搬到 iOS——通过 macOS 的 iPhone 镜像 点按、输入、滚动、截图，操作那些永远不会给你 API 的 App。仓库地址： github.com/leeguooooo/iphone-use。

⚠️ 各工具独立安装
家族成员共用哲学，但不是一个 monorepo。iphone-use、bitwarden-use、cookie-use 各自有独立仓库与安装步骤，请以各自 README 为准——本页只做导航，不代替它们的文档。
