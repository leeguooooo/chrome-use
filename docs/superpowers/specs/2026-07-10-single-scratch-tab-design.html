Single Scratch Tab Fix

2026-07-10

Problem

A new relay-backed session creates one owned about:blank target during target discovery. connect_auto_with_fresh_tab() then creates a second target unconditionally. Commands that do not navigate to a real URL can therefore leave two visible blank tabs.

Decision

Reuse the owned page returned by BrowserManager::connect_auto(). Create a fallback page only if the manager unexpectedly reports zero pages. This preserves session isolation and prevents navigation from replacing a user-owned tab.

Behavior

State after connectActionResult
One or more session pagesReuse the active owned pageNo duplicate blank tab
Zero pagesCreate one fallback blank pageConnection remains usable
Real URL opensKeep existing cleanupLeftover scratch pages are removed

Verification

A unit regression test covers the fresh-tab decision for zero and nonzero page counts. Existing Rust unit tests, formatting, and Clippy must remain clean.

No user-facing commands, flags, environment variables, or documented workflows change.
