# Nix flake Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Nix flake to chrome-use exposing a package, runnable app, dev shell, a `nix flake check`, and NixOS + home-manager modules.

**Architecture:** Build the pure-Rust `cli/` crate from source with `rustPlatform.buildRustPackage` (`buildAndTestSubdir = "cli"`), sourcing a filtered repo root so `build.rs`/`include_dir!` can reach `skills/`, `skill-data/`, and the extension manifest. Wire outputs with flake-parts. A single `nix/module.nix` produces both modules: the home-manager one registers the user-level native-messaging host by running the binary's own `chrome-use extension connect`; the NixOS one installs system-wide and optionally force-installs the extension via Chrome managed policy.

**Tech Stack:** Nix flakes, flake-parts, nixpkgs stable Rust (`rustPlatform.buildRustPackage`), home-manager + NixOS module systems.

> **DEVIATION FROM SPEC (approved-design used crane):** The crate is in a subdirectory (`cli/`) and `buildRustPackage` has `buildAndTestSubdir` built for exactly this, with no extra flake input. This plan uses `buildRustPackage` instead of crane. Everything else matches `docs/superpowers/specs/2026-07-23-nix-flake-design.md`. Consequence: no cached `cargoArtifacts` (deps recompile on a clean build); `nix flake check` runs build + clippy (tests need a real browser → CI, see Global Constraints).

## Global Constraints

- **Rust crate is at `cli/`** — no root `Cargo.toml`/workspace; `Cargo.lock` is `cli/Cargo.lock`. Build with `buildAndTestSubdir = "cli"`.
- **Build source must be the repo root, filtered.** `build.rs` reads `../extensions/ab-connect/manifest.json` and codegens from `cli/cdp-protocol/*.json`; `skills.rs` embeds `../skills` and `../skill-data`; `native/stream/http.rs` embeds `../packages/dashboard/out/` (created as a placeholder by `build.rs` at build time). The filtered source MUST include: `cli/` (minus `target/`), `skills/`, `skill-data/`, `extensions/ab-connect/manifest.json`.
- **Version:** `1.5.77` (from `cli/Cargo.toml`). Do not run `scripts/sync-version.js`.
- **License:** Apache-2.0 (`lib.licenses.asl20`). `mainProgram = "chrome-use"`.
- **`doCheck = false`** on the package — the crate's tests (`cli/tests`, `e2e_tests.rs`) need a real Chrome + network and cannot run hermetically. Tests stay a CI concern.
- **Constants (from `cli/src/connect.rs`):** Web Store extension id `knfcmbamhjmaonkfnjhldjedeobeafmk`; native-host command is `chrome-use extension connect`; `--keep-banner` skips restarting a running Chrome.
- **Supported systems:** `x86_64-linux aarch64-linux x86_64-darwin aarch64-darwin`.
- **Commit style:** Conventional Commits; end commit messages with `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

- [ ] **Step 1: Create `nix/package.nix`**

```nix
{ lib, rustPlatform }:

let
  root = ../.;

  # Keep only what the crate + its build.rs / include_dir! actually need.
  # A file is included only if the filter returns true for it AND every
  # ancestor directory, so directory entries must be kept explicitly.
  src = lib.cleanSourceWith {
    src = root;
    filter = path: _type:
      let
        rel = lib.removePrefix (toString root + "/") (toString path);
        keepTree = p: rel == p || lib.hasPrefix (p + "/") rel;
      in
        (keepTree "cli" && !(lib.hasInfix "/target/" ("/" + rel + "/")))
        || keepTree "skills"
        || keepTree "skill-data"
        || rel == "extensions"
        || rel == "extensions/ab-connect"
        || rel == "extensions/ab-connect/manifest.json";
  };
in
rustPlatform.buildRustPackage {
  pname = "chrome-use";
  version = "1.5.77";
  inherit src;

  cargoLock.lockFile = ../cli/Cargo.lock;
  buildAndTestSubdir = "cli";

  # Tests need a real Chrome + network; run them in CI, not the Nix sandbox.
  doCheck = false;

  meta = {
    description = "Drive your real, logged-in Chrome from any AI agent";
    homepage = "https://github.com/leeguooooo/chrome-use";
    license = lib.licenses.asl20;
    mainProgram = "chrome-use";
  };
}
```

- [ ] **Step 2: Create `flake.nix`**

```nix
{
  description = "chrome-use — drive your real, logged-in Chrome from any AI agent";

  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    flake-parts.url = "github:hercules-ci/flake-parts";
  };

  outputs = inputs@{ flake-parts, ... }:
    flake-parts.lib.mkFlake { inherit inputs; } {
      systems = [ "x86_64-linux" "aarch64-linux" "x86_64-darwin" "aarch64-darwin" ];

      perSystem = { pkgs, self', ... }: {
        packages.default = pkgs.callPackage ./nix/package.nix { };
        packages.chrome-use = self'.packages.default;

        apps.default = {
          type = "app";
          program = "${self'.packages.default}/bin/chrome-use";
        };

        formatter = pkgs.nixpkgs-fmt;
      };
    };
}
```

- [ ] **Step 1: Add checks**

In `flake.nix`, inside `perSystem`, add after `devShells.default`:

```nix

          build = self'.packages.default;

          clippy = self'.packages.default.overrideAttrs (old: {
            pname = "${old.pname}-clippy";
            nativeBuildInputs = (old.nativeBuildInputs or [ ]) ++ [ pkgs.clippy ];
            doCheck = true;
            checkPhase = ''
              cargo clippy --manifest-path cli/Cargo.toml --all-targets -- -D warnings
            '';
            installPhase = "touch $out";
          });
        };
```

### Task 4: NixOS + home-manager modules

**Files:**
- Create: `nix/module.nix`
- Modify: `flake.nix` (add flake-level `nixosModules` + `homeManagerModules`)

**Interfaces:**
- Consumes: `inputs.self.packages.${system}.default` (the Task 1 package).
- Produces: `nixosModules.default`, `homeManagerModules.default`. Both define `programs.chrome-use.{enable,package}`; home adds `{runOnActivation,connectFlags}`; nixos adds `{forceInstallExtension,extensionId}`.

- [ ] **Step 1: Create `nix/module.nix`**

```nix
# One factory → two modules. `mode` is "home" or "nixos".
{ flake, mode }:
{ config, lib, pkgs, ... }:

let
  cfg = config.programs.chrome-use;
  defaultPkg = flake.packages.${pkgs.system}.default;

  storeExtId = "knfcmbamhjmaonkfnjhldjedeobeafmk";
  updateUrl = "https://clients2.google.com/service/update2/crx";
  policy = { ExtensionInstallForcelist = [ "${cfg.extensionId};${updateUrl}" ]; };

  homeOpts = {
    runOnActivation = lib.mkOption {
      type = lib.types.bool;
      default = true;
      description = "Run `chrome-use extension connect` on activation to register the user-level native-messaging host.";
    };
    connectFlags = lib.mkOption {
      type = lib.types.listOf lib.types.str;
      default = [ "--keep-banner" ];
      description = "Flags for `chrome-use extension connect` (default skips restarting a running Chrome).";
    };
  };

  nixosOpts = {
    forceInstallExtension = lib.mkOption {
      type = lib.types.bool;
      default = false;
      description = "Force-install the chrome-use extension via Chrome managed policy (system-level; NixOS only).";
    };
    extensionId = lib.mkOption {
      type = lib.types.str;
      default = storeExtId;
      description = "Chrome Web Store id of the extension to force-install.";
    };
  };
in
{
  options.programs.chrome-use = {
    enable = lib.mkEnableOption "chrome-use (drive your real Chrome from an AI agent)";
    package = lib.mkOption {
      type = lib.types.package;
      default = defaultPkg;
      defaultText = lib.literalExpression "chrome-use flake package for this system";
      description = "The chrome-use package to install.";
    };
  } // lib.optionalAttrs (mode == "home") homeOpts
    // lib.optionalAttrs (mode == "nixos") nixosOpts;

  config = lib.mkIf cfg.enable (
    if mode == "home" then {
      home.packages = [ cfg.package ];
      home.activation = lib.optionalAttrs cfg.runOnActivation {
        chromeUseConnect = lib.hm.dag.entryAfter [ "writeBoundary" ] ''
          run ${lib.getExe cfg.package} extension connect ${lib.escapeShellArgs cfg.connectFlags} || true
        '';
      };
    } else {
      environment.systemPackages = [ cfg.package ];
      environment.etc = lib.mkIf cfg.forceInstallExtension {
        "opt/chrome/policies/managed/chrome-use.json".text = builtins.toJSON policy;
        "chromium/policies/managed/chrome-use.json".text = builtins.toJSON policy;
      };
    }
  );
}
```

## Notes for the implementer

- The single riskiest step is Task 1 Step 3 (`buildAndTestSubdir` + filtered source + git-dep hashes). Iterate on `nix build .#default -L` until green before moving on; every later task depends on the package building.
- If `nix build` reports git dependencies without hashes, that's expected for any non-crates.io dep — add `cargoLock.outputHashes` with the sha256 values Nix prints.
- The home-manager activation runs the real binary; `|| true` ensures a `home-manager switch` never fails just because no Chrome is installed yet. `--keep-banner` avoids restarting a running browser mid-switch. If `extension connect` turns out to block on a tty in a headless activation, set `programs.chrome-use.runOnActivation = false` and document running `chrome-use extension connect` manually.
