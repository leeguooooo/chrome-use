更新日志

每个版本改了什么、为什么改——这一页只留最近这批有实际影响的变化， 按新到旧排列。逐条 commit 级别的完整历史留在 GitHub Releases。

ℹ️ 完整历史：GitHub Releases
这一页只挑对日常使用有影响的条目。想看每个版本的完整 diff 和更早的历史，去 GitHub Releases。

v1.5.66

  • select 找不到匹配的 <option> 时不再默默选不中——现在会报错，并把所有可选项列出来。
  • 默认操作超时从 30 秒降到 25 秒，能在触发底层 socket 读超时之前干净地失败，而不是拖到那之后才报错。

v1.5.65

  • network route --set-json <dot.path>=<value>：对真实响应做结构化 JSON 字段编辑（创建或覆盖嵌套字段）；响应体不是 JSON 时自动退化为字符串替换。
  • 新增 chrome-use report [--open] [--json]：把本地、已脱敏的痛点日志打包成一份可直接粘贴的 GitHub issue——默认不开启，也从不自动上传。
  • MCP --tools all profile：24 个工具（12 个核心工具 + hover/select/screenshot/scroll/tabs/extract/expect/find/network_route/site/upload/download）。

v1.5.64

  • chrome-use mcp：给 Claude Desktop 这类只讲 MCP、不能跑 shell 命令的宿主起一个 stdio Model Context Protocol 服务器（12 工具核心 profile）。

v1.5.63

  • read [url]：把一个 URL（不传参数时是当前标签页）抓取成 agent 可读的 markdown/文本；支持 --llms / --outline / --filter / --raw。

v1.5.62

  • type --clear / --delay <ms> 现在能被正确解析成 flag。
  • find label 现在也匹配 aria-label 和 aria-labelledby（图标按钮、自定义控件不再找不到）。

v1.5.61

  • React renderer 修复：改成挑选真正的 react-dom renderer，不再硬编码 id 1——此前在 Next.js 16.3 Turbopack RSC 应用上会读到一棵空树。

v1.5.60

  • network route：在响应阶段编辑真实响应（覆盖状态码/响应头，对响应体做子串替换）。

v1.5.59

  • network route：通过 CDP 的 Fetch 域 mock 响应（fulfill）、改写发出去的请求（continue）——无需代理，无需证书，不注入 JS。

v1.5.58

  • chrome-use test：多账号 YAML 用例里支持按 case 切换账号（在某个 case 的步骤里写 account:）。
