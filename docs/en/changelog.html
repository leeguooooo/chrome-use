Changelog

What changed in each release, and why — this page keeps only the recent changes that actually affect day-to-day use, newest first. The full commit-level history lives in GitHub Releases.

ℹ️ Full history: GitHub Releases
This page curates the entries that matter for everyday use. For the complete diff of every release and older history, see GitHub Releases.

v1.5.66

  • select no longer silently selects nothing when no <option> matches — it now errors and lists every available option.
  • Default action timeout dropped from 30s to 25s, so it trips cleanly before the underlying socket read timeout instead of stalling past it.

v1.5.65

  • network route --set-json <dot.path>=<value>: structured JSON-field edits on the real response (create or overwrite nested fields); falls back to a string replace if the body isn't JSON.
  • New chrome-use report [--open] [--json]: packages the local, de-identified friction log into a paste-ready GitHub issue — opt-in, never auto-uploads.
  • MCP --tools all profile: 24 tools (the 12-tool core plus hover/select/screenshot/scroll/tabs/extract/expect/find/network_route/site/upload/download).

v1.5.64

  • chrome-use mcp: a stdio Model Context Protocol server (12-tool core profile) for MCP-only hosts like Claude Desktop that can't run a shell command.

v1.5.63

  • read [url]: fetch a URL — or, with no URL, the active tab — as agent-readable markdown/text; supports --llms / --outline / --filter / --raw.

v1.5.62

  • type --clear / --delay <ms> are now parsed correctly as flags.
  • find label now also matches aria-label and aria-labelledby — icon buttons and custom controls are no longer invisible to it.

v1.5.61

  • React renderer fix: pick the actual react-dom renderer instead of hardcoding id 1 — this was reading an empty tree on Next.js 16.3 Turbopack RSC apps.

v1.5.60

  • network route: edit the real response at the response stage (override status/headers, substring-replace the body).

v1.5.59

  • network route: mock responses (fulfill) and rewrite outgoing requests (continue) via the CDP Fetch domain — no proxy, no cert, no JS injection.

v1.5.58

  • chrome-use test: per-case account switching for multi-account YAML suites (write account: inside a case's steps).
